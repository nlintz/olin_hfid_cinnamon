Documentation of Limitations

User Data
  • The user profile only shows template profile (major and year of graduation)
  • User cannot see what courses they have already taken because we don’t have that data.
  • You cannot actually register using the site.
Database
  • The list of courses is incomplete
  • Data on website is manufactured
  • Prerequisite data is not included
  • Data specific to sections is not always available (for instance, if sections are taught by different professors).
  • This application is not actually connected to the Registrar in any way.
Search features
  • Search query does not autocomplete.
  • Search queries look through all of the course information.
  • Add courses function under course priority sidebar can give unexpected results.
Styling
  • Formatting and GUI are not styled perfectly.
  • Site navigation bar is not affixed to the top of the screen.
  • Site is not completely responsive to changing window sizes.
  • Courses sometimes show up in the same color on the calendar (since we’re currently only pulling from a list of eight colors).
  • Courses on course priority list are not color-coded to match courses in calendar.
  • Courses are not alphabetized in full course list.
